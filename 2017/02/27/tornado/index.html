tornado

发表于 2017-02-27 | 分类于 python开发 | | 阅读次数

Tornado是一个可扩展的非阻塞式web服务器及其相关工具的开源版本。tornado每秒可以处理若干连接。所以对于实时web服务来说，tornado是一个理想的web框架。

tornado介绍

tornado是使用python编写的一个强大的可扩展的web服务器。它在处理高流量时表现的足够强健，却在创建和编写的时有着足够的轻量级，并能够被用在大量的应用和开发中。

tornado特点

完备的web框架：与django/flask一样，tornado也提供了url路由映射；request上下文/基于模板的页面渲染技术等开发web应用的必备工具；
是一个高效的网络库，性能与twisted/gevent等底层python框架媲美；提供了异步I/O支持,超时事件处理。这使得tornado除了可以作为web应用服务器框架，还可以用来
做爬虫应用；
提供高效的内部http服务器：虽然其他python网络框架也提供了内部http服务器，但他们的http服务器由于性能原因只能用于测试环境。
完备的websocket支持，websocket是HTML5的一种新标准，实现了浏览器与服务器之间的双向实时通信。

异步及协程基础

协程是tornado中推荐的编码方式，使用协程可以开发出便捷/高效的异步代码。

python关键字yield

协程是tornado中进行异步I/O代码开发的方法。协程使用了python关键字yield将调用挂起和恢复执行。所以我们需要先学习下python中的yield。

迭代器

迭代器是访问集合内元素的一种方式。迭代器对象从集合的第一个元素开始访问，直到所有元素都被一一访问一遍后结束。迭代器不能回退，只能往前进行迭代。

详解：range()函数返回一个包含所制定元素的集合，而for语句将其封装成一个迭代器后访问。
使用iter（）可以将列表、集合转换为迭代器。

迭代器与普通python对象的区别是迭代器有一个next()方法，每次调用该方法可以返回一个元素。调用者可以通过不断调用next()方法来逐个访问集合元素。调用者可以一直调用next()
方法来访问迭代器，直到next()返回stopIteration异常来表示迭代已经完成。

使用yield

迭代器在python编程中的使用范围很广，那么开发者如何定制自己的迭代器呢？答案是使用yield关键字。调用任何定义中包含yield关键字的函数都不会执行该函数，而会获得一个
对应该函数的迭代器。

每次调用迭代器的next（）函数，并返回yield的结果作为迭代返回元素。当迭代器函数return时，迭代器会抛出stopIteration异常使迭代终止。
