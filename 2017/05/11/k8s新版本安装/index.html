k8s新版本安装（1.6.2）

发表于 2017-05-11 | 分类于 k8 | | 阅读次数

国内镜像源配置

docker.repo
cat >> /etc/yum.repos.d/docker.repo <<EOF
[docker-repo]
name=Docker Repository
baseurl=http://mirrors.aliyun.com/docker-engine/yum/repo/main/centos/$releasever/
enabled=1
gpgcheck=0
EOF

kubernetes yum源
cat >> /etc/yum.repos.d/kubernetes.repo <<EOF
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64/
enabled=1
gpgcheck=0
EOF

安装docker

查看docker版本
yum list docker-engine –showduplicates

Kubernetes 1.6还没有针对docker 1.13和最新的docker 17.03上做测试和验证，所以这里安装Kubernetes官方推荐的Docker 1.12版本。
yum install -y docker-engine-1.12.6-1.el7.centos.x86_64

修改系统参数

根据官方文档Installing Kubernetes on Linux with kubeadm中的Limitations小节中的内容，对各节点系统做如下设置:

创建/etc/sysctl.d/k8s.conf文件,添加如下内容:

‘’’
cat >>/etc/sysctl.d/k8s.conf<< EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
EOF
‘’’

执行 sysctl -p使其生效。

安装kubeadm和kubelet

查看kubeadm, kubelet, kubectl, kubernets-cni的最新版本:
yum list kubeadm –showduplicates
kubeadm.x86_64 1.6.1-0 kubernetes
kubeadm.x86_64 1.6.1-0 mritdrepo
kubeadm.x86_64 1.6.2-0 kubernetes
kubeadm.x86_64 1.6.2-0 mritdrepo

yum list kubelet –showduplicates
kubelet.x86_64 1.6.2-0 kubernetes
kubelet.x86_64 1.6.2-0

yum install -y kubernetes-cni-0.5.1-0.x86_64 kubelet-1.6.2-0.x86_64 kubectl-1.6.2-0.x86_64 kubeadm-1.6.2-0.x86_64

启用kubelet服务

systemctl enable kubelet.service

初始化集群

准备MasterConfiguration文件
‘’’
[root@ucloud-test-bd-k8s-master1 ~]# cat k8s_master_config.yaml
apiVersion: kubeadm.k8s.io/v1alpha1
kind: MasterConfiguration
api:
advertiseAddress: 10.10.17.59
networking:
dnsDomain: cluster.local
etcd:
endpoints:

  • http://10.10.29.103:2380
  • http://10.10.48.162:2380
  • http://10.10.9.68:2379
    kubernetesVersion: v1.6.2
    ‘’’

修改docker私服地址

‘’’
cat /usr/lib/systemd/system/docker.service
[Unit]
Description=Docker Application Container Engine
Documentation=http://docs.docker.com
After=network.target
Wants=docker-storage-setup.service
Requires=docker-cleanup.timer

[Service]
Type=notify
NotifyAccess=all
EnvironmentFile=-/etc/sysconfig/docker
EnvironmentFile=-/etc/sysconfig/docker-storage
EnvironmentFile=-/etc/sysconfig/docker-network
Environment=GOTRACEBACK=crash
Environment=DOCKER_HTTP_HOST_COMPAT=1
Environment=PATH=/usr/libexec/docker:/usr/bin:/usr/sbin
ExecStart=/usr/bin/dockerd-current \
–add-runtime docker-runc=/usr/libexec/docker/docker-runc-current \
–default-runtime=docker-runc \
–exec-opt native.cgroupdriver=systemd \
–userland-proxy-path=/usr/libexec/docker/docker-proxy-current \
–insecure-registry 10.10.132.89 \
$OPTIONS \
$DOCKER_STORAGE_OPTIONS \
$DOCKER_NETWORK_OPTIONS \
$ADD_REGISTRY \
$BLOCK_REGISTRY \
$INSECURE_REGISTRY
ExecReload=/bin/kill -s HUP $MAINPID
LimitNOFILE=1048576
LimitNPROC=1048576
LimitCORE=infinity
TimeoutStartSec=0
Restart=on-abnormal
MountFlags=slave

[Install]
WantedBy=multi-user.target
‘’’

进行初始化操作：
‘’’
systemctl stop kubelet;
docker rm -f -v $(docker ps -q);
find /var/lib/kubelet | xargs -n 1 findmnt -n -t tmpfs -o TARGET -T | uniq | xargs -r umount -v;
rm -r -f /etc/kubernetes /var/lib/kubelet /var/lib/etcd
export ECTD_API=3
etcdctl get –prefix /registry/ –only-keys
systemctl start kubelet;
‘’’
kubeadm init –config=k8s_master_config.yaml

出现token说明Master初始化完成，使用kubeadm初始化的Kubernetes集群在Master节点上的核心组件：kube-apiserver,kube-scheduler,
kube-controller-manager是以静态Pod的形式运行的。
ls -l /etc/kubernetes/manifests/
etcd.yaml
kube-apiserver.yaml
kube-controller-manager.yaml
kube-scheduler.yaml
如果想修改kubernetes组件的参数, 可以修改对应组件的yaml配置文件, 然后重启kubelet服务使之生效

systemctl restart kubelet

都成功之后运行如下命令，先不要问为什么。
sudo cp /etc/kubernetes/admin.conf $HOME/
sudo chown $(id -u):$(id -g) $HOME/admin.conf
export KUBECONFIG=$HOME/admin.conf

部署pod 网络

kubectl create -f kube-flannel-rbac.yml
kubectl create -f kube-flannel.yml

删除网络
ip addr del 10.16.0.0/32 dev flannel.1
ip addr del 10.16.1.0/32 dev flannel.1

vim /etc/systemd/system/kubelet.service.d/10-kubeadm.conf
Environment=”KUBELET_CGROUP_ARGS=–cgroup-driver=cgroupfs”

节点加入到集群

kubeadm join –token c9b51c.416daf3a9bf8ebcd 10.10.17.59:6443
如果忘记token，可以使用如下命令：
kubeadm token list

总结下1.6版本查询key值方法：1、export ETCDCTL_API=3；2、etcdctl get –prefix / –keys-only

kubectl delete po xxx –grace-period=0
