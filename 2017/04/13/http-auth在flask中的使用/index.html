对于初学者来说，一个真正的Web服务应该由一个真实的数据库支持。我们使用的内存数据结构在功能上非常有限，不应该用于实际应用。
可以进行改进的另一个领域是处理多个用户。如果系统支持多个用户，则由客户端发送的身份验证凭据可用于获取用户特定的待办事项列表。

在这样的系统中，我们将有第二个资源，这将是用户。用户资源上的POST请求将表示为该服务注册的新用户。GET请求将用户信息返回给客户端。
提交请求将更新用户信息，可能更新电子邮件地址。删除请求将删除用户帐户。检索任务列表的GET请求可以通过两种方式扩展。首先，这个要求可以随意分
页参数，使客户端可以请求列表的一部分。使这个函数更有用的另一种方法是允许按一定的标准过滤。
例如，一个客户端可能只想看到完成的任务，或者只使用带有字母A的标题的任务。所有这些元素都可以作为参数添加到URL中。
