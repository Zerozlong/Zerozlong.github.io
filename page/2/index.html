事件冒泡

IE的事件流叫做事件冒泡，即事件开始时由最具体的元素接收，然后逐级向上传播到较为不具体的节点。以下面的HTML页面为例子。
当你点击div元素，那么这个click事件会按照如下顺序传播：1、div；2、body；3、html；4、document。也就是说click事件首先在
div元素上发生，而这个元素就是我们单击的元素。然后，click事件沿dom数向上传播，在每一个节点上都会发生，直至传播到document对象。

事件

事件是用来处理响应的一个机制，这个响应可以来自于用户(点击, 鼠标移动, 滚动), 也可以来自于浏览器。
下面的链接描述了所有事件https://developer.mozilla.org/en-US/docs/Web/Events；
常用例子, 给按钮添加一个点击的事件。
1、获得按钮 var loginButton = document.querySelector(‘#id-button-login’)
2, 声明一个函数, 用于在按钮点击后执行
var clicked = function() {
alert(‘按钮被点击到了’)
}
3, 添加事件, 使用 addEventListener 函数.它有两个参数,
第一个是事件的名字, ‘click’ 表示点击,第二个是事件发生后会被自动调用的函数
loginButton.addEventListener(‘click’, clicked)
添加完成, 可以自己在浏览器试试, 记得打开 console.

js的序列化和反序列化

有一个常见的需求是在 字典/数组 和字符串 之间相互转换，这个过程叫做 序列化/反序列化在 js 中, 序列化使用 JSON 数据格式。
全称 JavaScript Object Notation (js 对象标记)。这个格式已经是现在用于互联网数据交换的事实标准格式了；python 也有内置的标准库进行这种转换。
js 中转换方式如下。注意 typeof 语句可以查看一个变量的类型
var s = JSON.stringify([1, 2, 3, 4])
log(‘序列化后的字符串’, typeof s, s)
var a = JSON.parse(s)
在 js 中, 由于历史遗留的 bug，typeof [] 会显示 object, 这是正常的
log(‘反序列化后的数组’, typeof a, a)

gpu与docker(一)

发表于 2017-10-19 | 分类于 gpu | | 阅读次数

参考资料：https://github.com/NVIDIA/nvidia-docker/wiki/Motivation

集成gpu应用程序的优势

可复制的版本
易于部署
隔离各个设备
跨异构驱动程序/工具包环境运行
只需要安装NVIDIA驱动程序

背景

Docker容器通常用于在多台机器上无缝部署基于CPU的应用程序。有了这个用例，容器可以看做是平台无关和硬件无关的。
使用NVIDIA GPU显然不是这样，因为它使用专门的硬件，并且需要安装NVIDIA驱动程序。因此，Docker Engine本身并不支持
具有NVIDIA GPU的容器。

为了解决这个问题，出现的一个早期解决方案是在容器内完全重新安装NVIDIA驱动程序，然后/dev/nvidia0在启动容器时
传递与NVIDIA GPU（例如）对应的字符设备。但是，该解决方案是脆弱的：主机驱动程序的版本必须与容器中安装的驱动程序版本完全匹配。
Docker镜像无法共享，必须在当前机器上进行构建，从而打破了Docker的主要优势之一。

为了使Docker镜像可移植，同时仍然利用NVIDIA GPU，所使用的解决方案是使用nvidia-docker；该方案让镜像与NVIDIA驱动程序无关。
当将容器运行到目标机器上时候，安装所需的字符设备和驱动程序文件。

另外一个docker

由于该解决方案处理细节细节，与普通Docker容器用例完全不同，为方便起见提供了两个工具： alternative Docker CLI和Docker Engine plugin。
虽然我们明白使用辅助命令行工具可能令人沮丧，但我们尽量保持与Docker的理念尽可能接近。

我们希望在未来Docker将增强其插件系统，以便在命令行中更容易地添加设备和参数，使我们的工具过时。
