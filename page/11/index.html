再次理解下yield表达式

生成器是通过一个或多个yield表达式构成的函数，每一个生成器都是一个迭代器（但是迭代器不一定是生成器）。
如果一个函数包含yield关键字，这个函数就会变为一个生成器。
生成器并不会一次返回所有结果，而是每次遇到yield关键字后返回相应结果，并保留函数当前的运行状态，等待下一次的调用。
由于生成器也是一个迭代器，那么它就应该支持next方法来获取下一个值。

Tornado是一个可扩展的非阻塞式web服务器及其相关工具的开源版本。tornado每秒可以处理若干连接。所以对于实时web服务来说，tornado是一个理想的web框架。

tornado介绍

tornado是使用python编写的一个强大的可扩展的web服务器。它在处理高流量时表现的足够强健，却在创建和编写的时有着足够的轻量级，并能够被用在大量的应用和开发中。

tornado特点

完备的web框架：与django/flask一样，tornado也提供了url路由映射；request上下文/基于模板的页面渲染技术等开发web应用的必备工具；
是一个高效的网络库，性能与twisted/gevent等底层python框架媲美；提供了异步I/O支持,超时事件处理。这使得tornado除了可以作为web应用服务器框架，还可以用来
做爬虫应用；
提供高效的内部http服务器：虽然其他python网络框架也提供了内部http服务器，但他们的http服务器由于性能原因只能用于测试环境。
完备的websocket支持，websocket是HTML5的一种新标准，实现了浏览器与服务器之间的双向实时通信。

迭代器

迭代器是访问集合内元素的一种方式。迭代器对象从集合的第一个元素开始访问，直到所有元素都被一一访问一遍后结束。迭代器不能回退，只能往前进行迭代。

详解：range()函数返回一个包含所制定元素的集合，而for语句将其封装成一个迭代器后访问。
使用iter（）可以将列表、集合转换为迭代器。

迭代器与普通python对象的区别是迭代器有一个next()方法，每次调用该方法可以返回一个元素。调用者可以通过不断调用next()方法来逐个访问集合元素。调用者可以一直调用next()
方法来访问迭代器，直到next()返回stopIteration异常来表示迭代已经完成。

使用yield

迭代器在python编程中的使用范围很广，那么开发者如何定制自己的迭代器呢？答案是使用yield关键字。调用任何定义中包含yield关键字的函数都不会执行该函数，而会获得一个
对应该函数的迭代器。

每次调用迭代器的next（）函数，并返回yield的结果作为迭代返回元素。当迭代器函数return时，迭代器会抛出stopIteration异常使迭代终止。

Python网络框架的概念

目前Python编程的网络框架已经多达几十个，逐个学习他们显然不现实。如下文章只介绍框架的常用知识。

网络框架及MVC架构

网络框架是指这样的一组python包，它能够使开发者专注于网站应用业务逻辑的开发，而无需处理网络应用底层的协议/线程/进程等方面。
好处：能够大大提高开发者开发的效率，同时提高网络应用程序的质量。

在目前python的几十个开发框架中，几乎所有的全栈网络框架都强制或引导开发者使用MVC架构开发WEB应用。所谓全栈网络框架，是指除了封装网络和
线程操作，还提供http栈，数据库读写管理、html模板引擎等一系列功能的网络框架。如django、flask、tornado。而twisted更专注于网络底层的高性能
封装而不提供渲染html模板引擎等界面功能，所以不能称之为全栈框架。

网络上有许多关于MVC的说法，MVC大概把应用系统分为3层。

模型（model）

用于封装与应用程序的业务逻辑相关的数据及对数据处理的方法，是web应用程序中用于处理应用程序的数据逻辑部分，model只提供功能性的接口，通过这些接口
可以获取model的所有功能。model不依赖于view和controller，它们可以在任何时候调用model访问数据。有些model还提供了事件通知机制，为在其上的注册过的view
或者controller提供实时数据更新。

视图（view）

负责数据的展示和呈现，view是对用户的直接输出，mvc中的一个modle通常为多个view提供服务。为了获取model的实时更新数据，view应该尽早注册到model中。

控制器（controller）

负责从用户端收集用户的输入，可以看成提供View的反向功能。当用户的输入导致view发生变化时，这种变化必须是通过model反映给view的。在mvc架构下，controller一般
不能与view直接通信，这样提高了业务数据的一致性，即以model作为数据中心。

python ORM库介绍

发表于 2017-02-24 | 分类于 python开发 | | 阅读次数

python中提供orm支持的组件有很多，每个组件的应用领域稍有区别，但是数据操作的理论原理是相同的。下面对比较著名的
python数据库的orm框架介绍如下。

SQLALchemy

python中最成熟的ORM框架，资源和文档都很丰富。大多数Python web框架都有很好的支持。能够胜任大多数python开发的场合；

Django ORM

python世界中大名鼎鼎的Django web框架独用的ORM技术。DJANGO是一个大而全的框架，这使得其灵活性大大降低。其他python框架
可以随意更换orm，但django不能这样做，因为django内置的model是用django内置orm实现的。

peewee

小强灵活，是一个轻量级的ORM。peewee是基于sqlchemy内核开发的，整个框架只有一个文件构成。peewee提供了对多种数据库访问方式，
如sqllite/mysql/postsql，适用于功能简单的小型网站。

storm

中型的ORM库，比sqlalchem和django等轻量，比peewee更丰富。storm要求开发者编写数据表的DDL代码，而不能直接从数据表定义中自生成表定义。

sqlobject

与sqlalchemy相似，也是一套大而全的ORM。sqlobject的特点是其设计借鉴了ruby on rails的ActiveRecord模型，使得熟悉RUBY的开发者能够快速上手。
